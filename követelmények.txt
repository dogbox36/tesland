2) GitHub: lépésenkénti nyomkövetés (nem opcionális)

A teljes fejlesztés GitHub repo-ban, lépésenként legyen rögzítve. Kötelező:

Minden érdemi változás külön commit (ne 1 nagy “final” commit)

Kötelező branch/PR munkafolyamat:

main = stabil

develop = integráció

feature branchek: feat/..., fix/..., chore/...

minden feature PR-on keresztül menjen develop-ba

Kötelező commit convention:

feat: ..., fix: ..., chore: ..., docs: ..., test: ..., refactor: ...

Minden PR-hoz:

leírás: mi változott, hogyan tesztelted, screenshot ha UI

checklista: lint/test/build futott

Minden mérföldkőnél Git tag:

v0.1-spec, v0.2-auth, v0.3-shop, v0.4-booking, v0.5-mobile, v1.0-mvp

Kötelező CHANGELOG.md (Keep a Changelog stílus) + verziózás (SemVer)

TECH STACK ELVÁRÁS (javasolj, de legyen modern és stabil)

Preferált irány:

Web: Next.js (App Router) + TypeScript + Tailwind + SEO

API: NestJS vagy Fastify/Express TypeScript + OpenAPI

DB: PostgreSQL + Prisma (vagy TypeORM)

Cache/Queue: Redis (értesítések, booking lock, job queue)

File storage: S3 kompatibilis (képek, csatolmányok)

Mobile: React Native (Expo) vagy Flutter (indokold)

Auth: JWT + refresh token + optional OAuth later

Payments: 2 integráció (pl. Stripe + magyar szolgáltató), absztrakcióval

Notifications: email + push (FCM/APNs)

CI/CD: GitHub Actions (lint/test/build)

BOOKING MOTOR – KÖTELEZŐ LOGIKA
